# Prompt 10: Core API And CLI

Date: 2026-06-16

## Scope

Implemented the thin public core conversion API and wired the `argparse` CLI
commands `convert`, `inspect` and `validate` to core functions. No live
download flows, backend service, database, Click or Typer dependency was
added.

## Public API And CLI Command Summary

Core conversion API:

- Added `dwca_cloud_geospatial.conversion.convert_dwca_archive`.
- Added `ConversionOptions`, `ConversionResult` and `ConversionError`.
- Added output format constants `FLATGEOBUF_FORMAT`, `GEOPARQUET_FORMAT` and
  `SUPPORTED_OUTPUT_FORMATS`.
- Exported the conversion API from `dwca_cloud_geospatial`.

Conversion behavior:

- Default output format is FlatGeobuf at `exports/occurrences.fgb`.
- Explicit GeoParquet output is selected with output format `geoparquet` and
  writes `data/occurrences.parquet`.
- Passing both `flatgeobuf` and `geoparquet` writes both geospatial outputs
  from the same accepted normalized record set.
- Core conversion calls the existing occurrence row reader, normalization API,
  selected writer APIs and then `write_bundle_metadata`.
- CLI handlers do not duplicate parser, writer, manifest or rejected-report
  logic.
- FlatGeobuf dependency errors and GeoParquet dependency errors are surfaced
  as actionable `ConversionError` messages.
- FlatGeobuf writer warnings, including `large_indexed_flatgeobuf_write`,
  remain non-fatal and are preserved through bundle metadata.

CLI commands:

- `dwca-cloud-geospatial inspect [--json] <archive>` remains a thin wrapper
  around `dwca_cloud_geospatial.inspection.inspect_dwca`.
- `dwca-cloud-geospatial convert <archive> <output>` writes the default
  FlatGeobuf bundle.
- `dwca-cloud-geospatial convert <archive> <output> --format geoparquet`
  writes explicit GeoParquet output.
- Repeating `--format` selects multiple outputs.
- `dwca-cloud-geospatial convert ... --overwrite` is required to replace an
  existing output path.
- `dwca-cloud-geospatial validate [--json] <bundle>` calls
  `dwca_cloud_geospatial.validation.validate_output_bundle` directly and
  exits non-zero only when `.has_errors` is true.

## Failure Behavior

- Checklist/Taxon archives remain valid for `inspect` and `inspect --json`.
- `convert` rejects non-occurrence archives using the occurrence row reader's
  `missing_occurrence_core` diagnostic.
- Conversion fails fast when the Occurrence core lacks coordinate terms.
- Conversion fails when normalization leaves no accepted records.
- Optional conversion warnings do not fail conversion by themselves.

```bash
.venv/bin/dwca-cloud-geospatial convert tests/fixtures/dwca/minimal_occurrence/valid /private/tmp/dwca_prompt10_default
```

Result: exit `1` after the output directory already existed, with:

```text
Conversion failed: Output path already exists: /private/tmp/dwca_prompt10_default. Pass --overwrite to replace it.
```

Command:

```bash
.venv/bin/dwca-cloud-geospatial convert tests/fixtures/dwca/minimal_occurrence/valid /private/tmp/dwca_prompt10_default --overwrite
```

Result: exit `0`; wrote default FlatGeobuf bundle with `1` accepted record,
`0` rejected records and manifest at
`/private/tmp/dwca_prompt10_default/manifest.json`.
